# 27-4 Criar sistema de cenários múltiplos

[Voltar para lista de tarefas](./tasks.md)

## Description
Implementar geração de cenários de previsão (conservador, realista e otimista) com base no baseline da 27-2 e nos sinais da 27-3.

## Requirements
- [x] Definir metodologia para P10/P50/P90 (ou equivalente).
- [x] Produzir saídas de cenários no mesmo contrato do forecast base.
- [x] Permitir parametrização de horizonte e variáveis de sensibilidade.
- [x] Documentar premissas e limites dos cenários.

## Implementation Plan
1. Definir estratégia estatística para bandas de cenário.
2. Implementar gerador de cenários sobre previsão base.
3. Padronizar payload de saída para consumo interno.
4. Validar coerência entre os três cenários.

## Premissas e limites (v1)

- Cenários calculados sobre o forecast base com fatores multiplicativos fixos:
  - conservador: `0.9`
  - realista: `1.0`
  - otimista: `1.1`
- Para métricas de volume (`leads`, `spend`, `impressions`, `clicks`) os cenários usam o **total**.
- Para métricas de eficiência (`ctr`, `cpl`) os cenários usam a **média**.
- O modelo de cenários fica identificado em `metadata.scenarioModel = forecast_scenarios_v1`.
