# 27-5 Desenvolver modelos específicos por campanha

[Voltar para lista de tarefas](./tasks.md)

## Description
Evoluir o forecasting para suportar modelos segmentados por tipo de campanha, reduzindo erro em grupos heterogêneos.

## Requirements
- [x] Definir regra de segmentação por campanha/tipo.
- [x] Implementar treinamento separado por segmento elegível.
- [x] Definir fallback para segmentos com pouco histórico.
- [x] Registrar métricas comparativas contra modelo global.

## Implementation Plan
1. Mapear segmentos com volume mínimo de dados.
2. Implementar pipeline de treino segmentado.
3. Criar estratégia de fallback para cold-start.
4. Comparar performance entre global vs segmentado.

## Verification
- [x] Segmentos são treinados apenas quando cumprem critérios mínimos.
- [x] Fallback funciona para segmentos insuficientes.
- [x] Métricas comparativas documentadas.

## Test Plan
- **Objective:** validar ganho de precisão com segmentação.
- **Scope:** treino/inferência por segmento e fallback.
- **Env/Setup:** dados históricos com metadados de campanha.
- **Scenarios:** segmento robusto e segmento com pouco dado.
- **Criteria:** comportamento estável e comparativo reproduzível.

## Premissas e limitações (fase atual)

- Segmentação inicial habilitada por filtro de entrada na API (`campaignId` e `adsetId`).
- O baseline usa o mesmo pipeline de forecast, alterando apenas o subconjunto de treino/inferência.
- Quando não há histórico no segmento selecionado, o fallback operacional retorna série zerada estável (sem erro de execução).
- Métricas comparativas são expostas em `metadata.segmentationComparison` com `segmentedTotal`, `globalTotal` e `deltaPercent`.
