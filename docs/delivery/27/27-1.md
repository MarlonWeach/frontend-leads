## Requirements
- [ ] Decisão documentada de plataforma temporal (ex.: Timescale em Postgres dedicado vs extensão/hypertables no ambiente Supabase já usado pelo projeto).
- [ ] Esquema inicial de tabelas ou views materializadas para séries temporais de métricas relevantes (leads, CPL, spend, conversões conforme disponibilidade).
- [ ] Pipeline de ingestão ou agregação (batch ou incremental) com janelas definidas e idempotência onde aplicável.
- [ ] Contrato mínimo de qualidade de dados (granularidade, timezone, chaves de adset/campanha) para consumo pelo motor de features.
- [ ] Documentação técnica do desenho (diagrama ou texto) linkada nesta task ao concluir a fase de desenho.

## Implementation Plan
1. Inventariar fontes atuais (`adset_insights`, metas, APIs) e latência aceitável para previsão.
2. Comparar opções de armazenamento temporal e custo operacional; registrar decisão no repositório (task ou doc existente do PBI 27).
3. Especificar e implementar migração ou jobs de agregação alinhados à decisão.
4. Validar leitura ponta a ponta (ex.: query ou job de smoke) antes de encerrar a task.
