# 27-8 Implementar sistema de alertas preditivos

[Voltar para lista de tarefas](./tasks.md)

## Description
Implementar alertas automáticos baseados em previsões para antecipar risco de queda de leads, piora de CPL ou desvios relevantes de tendência.

## Requirements
- [x] Definir regras de alerta com thresholds configuráveis.
- [x] Gerar alertas com contexto e recomendação inicial.
- [x] Integrar com mecanismo de notificação já existente no projeto.
- [x] Evitar spam com cooldown/rate limit.

## Implementation Plan
1. Definir catálogo de alertas preditivos prioritários.
2. Implementar motor de avaliação de regras sobre previsões.
3. Integrar persistência/entrega de alertas.
4. Validar cooldown e deduplicação.

## Verification
- [x] Regras disparam alertas corretos em cenários de teste.
- [x] Cooldown evita alertas duplicados excessivos.
- [x] Alertas carregam contexto suficiente para ação.

## Test Plan
- **Objective:** validar antecipação de risco por alertas.
- **Scope:** cálculo de regras e emissão de alertas.
- **Env/Setup:** previsões e infraestrutura de alerta disponíveis.
- **Scenarios:** risco alto, risco moderado e cenário normal.
- **Criteria:** alertas relevantes, não ruidosos e acionáveis.

## Premissas e limitações (fase atual)

- Alertas preditivos estão disponíveis em `metadata.predictiveAlerts` no endpoint de forecast.
- Regras atuais cobrem tendência de queda em volume, piora em CPL e degradação de acurácia.
- Alertas são persistidos na tabela `alerts` (mecanismo existente) e ficam disponíveis para consumo no dashboard.
- Cooldown de 60 minutos por `adset_id + title` evita duplicação excessiva no mesmo horizonte.
