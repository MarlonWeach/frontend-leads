# 34-4 Aplicar políticas RLS por grupo de tabelas

## Description
Tratar sugestões `rls_enabled_no_policy` para tabelas com RLS habilitado e sem policies, aplicando política explícita de deny-by-default para `anon` e `authenticated` sem ampliar superfície de acesso.

## Requirements
- [x] Cobrir tabelas reportadas pelo Advisor com RLS habilitado e sem policy.
- [x] Não abrir leitura/escrita indevida para `anon`/`authenticated`.
- [x] Manter execução idempotente para ambientes já parcialmente configurados.

## Implementation Plan
1. Criar lista de tabelas alvo do warning.
2. Para cada tabela existente:
   - garantir `ENABLE ROW LEVEL SECURITY`;
   - criar policies de deny-by-default apenas se não houver policy prévia.
3. Preservar comportamento de backend via `service_role`.
