## Requirements
- [x] Permitir limpar recomendações ativas da conta piloto em um único clique, preservando trilha de auditoria.
- [x] Permitir gerar nova rodada sobrescrevendo (expirando) recomendações ativas anteriores.
- [x] Expor controles explícitos na UI `/dashboard/recomendacoes`.
- [x] Manter proteção de acesso por conta piloto autorizada.

## Implementation Plan
1. Adicionar endpoint para descarte em massa de recomendações `active` com registro de decisão.
2. Estender endpoint de geração com flag de sobrescrita para expirar ativas antes de gerar.
3. Incluir botões no painel para os dois fluxos e feedback visual de sucesso/erro.
4. Validar integridade com typecheck e leitura de lints.

## Test Plan
Objetivo: validar operação segura do piloto em ambiente real com ações em lote.

Escopo:
- Conta piloto autorizada em `/dashboard/recomendacoes`.
- Recomendações com status `active`.

Cenários:
1. Clicar em `Limpar ativas` e confirmar:
   - status das recomendações muda para `discarded`;
   - cria entradas em `optimization_recommendation_decisions` com `action=discard`.
2. Clicar em `Gerar sobrescrevendo ativas`:
   - recomendações `active` anteriores passam para `expired`;
   - nova rodada é gerada na sequência.
3. Clicar em `Gerar sugestões (motor v1)` sem sobrescrita:
   - mantém comportamento anterior (não substitui ativas existentes).
4. Conta fora do piloto:
   - endpoints retornam `403` com `META_ACCOUNT_FORBIDDEN`.
