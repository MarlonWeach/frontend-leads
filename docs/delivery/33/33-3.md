# 33-3 Implementar cadastro e edição de metas na UI

[Voltar para lista de tarefas](./tasks.md)

## Description
Implementar a experiência de cadastro e edição de metas por adset na página `/metas`, com persistência real nas rotas de API já existentes e atualização imediata do dashboard após salvar.

## Requirements
- [x] Permitir abrir modal de configuração em cada card de adset.
- [x] Salvar meta via API (`POST /api/goals` para criação, `PUT /api/goals/[adset_id]` para edição).
- [x] Permitir edição de campos operacionais principais.
- [x] Atualizar visualização após salvar sem recarregar a página.

## Implemented Scope
- Modal de meta na `AdsetGoalCard` com campos:
  - `budget_total`
  - `cpl_target`
  - `volume_contracted`
  - `volume_captured` (manual)
  - `contract_start_date`
  - `contract_end_date`
- Persistência mantendo o fluxo existente da API:
  - criação (quando não existe meta);
  - atualização (quando já existe meta).
- Feedback visual de sucesso/erro no modal.
- Refresh real do dashboard após salvar meta.

## Technical Notes
- Removido padrão frágil de callback com `Object.assign` no `onSave`.
- Hook `useAdsetGoals` ajustado para `refresh` efetivo via `reloadToken`.
- Página de overview passou a usar `refresh()` após edição para refletir dados mais recentes.
