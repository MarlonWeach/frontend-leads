# 33-1 Diagnosticar e restaurar acesso da página `/metas`

[Voltar para lista de tarefas](./tasks.md)

## Description
Diagnosticar por que a página de metas não estava acessível no fluxo esperado e restaurar acesso estável pela URL `/metas`.

## Requirements
- [x] Identificar rota real implementada para metas.
- [x] Garantir acesso pela URL amigável `/metas`.
- [x] Garantir proteção de autenticação para a nova rota.
- [x] Ajustar navegação principal para o caminho canônico.

## Root Cause
- A página existente estava em `/dashboard/metas`.
- Não havia rota explícita em `/metas`, causando falha para acesso direto por URL esperada.

## Implemented Solution
1. Criada rota `app/metas/page.tsx` com redirecionamento para `/dashboard/metas`.
2. Middleware atualizado para tratar `/metas` como rota privada.
3. Navegação no `MainLayout` atualizada para usar `/metas`.

## Verification

- [x] `/metas` redireciona corretamente para `/dashboard/metas`.
- [x] `/metas` exige autenticação via middleware.
