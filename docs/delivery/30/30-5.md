# 30-5 Hardening de sessão e segredos

[Voltar para lista de tarefas](./tasks.md)

## Description
Endurecer a sessão server-side (cookies + renovação) e reduzir superfície de ataque no fluxo de autenticação, garantindo expiração e renovação controlada sem expor segredos no client.

## Requirements
- [x] Expiração explícita para access token (curta) e refresh token (maior).
- [x] Renovação segura via refresh token no servidor.
- [x] Logout limpa cookies localmente.
- [x] Endpoints de auth com `Cache-Control: no-store`.
- [x] Controles básicos anti-CSRF (same-origin) no login/logout/refresh.
- [x] Segredos permanecem apenas server-side (nenhum `service_role` no client).

## Implementation Plan
- Centralizar TTLs de cookies em utilitário de sessão.
- Implementar `/api/auth/refresh` usando `refreshSession`.
- Atualizar `/api/auth/session` para tentar refresh quando access token expirar.
- Adicionar validação de `Origin`/`sec-fetch-site` em endpoints sensíveis.
- Incluir `/api/auth/*` no matcher do middleware para aplicar rate limit global existente.
