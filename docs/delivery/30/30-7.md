# 30-7 Teste E2E de CoS de autenticação (obrigatório)

[Voltar para lista de tarefas](./tasks.md)

## Description
Validar ponta a ponta os critérios de autenticação e autorização mínimas para o dashboard: acesso negado sem sessão, sessão válida liberando rotas privadas e logout revertendo o acesso.

## Requirements
- [x] Acesso a páginas privadas sem cookies deve redirecionar para `/login`.
- [x] Com cookies de sessão, `/login` deve redirecionar para `/dashboard`.
- [x] Criação de sessão válida deve liberar acesso ao `/dashboard`.
- [x] Logout deve limpar cookies e voltar a bloquear acesso.

## Implementation Plan
- Criar `e2e/tests/auth.spec.ts`.
- Atualizar `playwright.config.ts` para incluir o novo spec.
- Adicionar endpoint dedicado de seed/clear de sessão para E2E (`/api/auth/test/session`) habilitado somente com flag `PLAYWRIGHT_AUTH_E2E=1`.
- Usar `page.request` para seed de sessão e validar comportamento real de middleware.
- Usar mocks existentes de Supabase/Meta para evitar dependência de dados reais ao renderizar `/dashboard`.

## Verification
- [x] `/dashboard` sem sessão -> redirect para `/login`.
- [x] `/login` com cookie -> redirect para `/dashboard`.
- [x] Sessão ativa (seed E2E) -> `/dashboard`.
- [x] Logout -> limpar cookies -> `/login`.
