# 30-8 Implementar provisionamento de usuários apenas por admin

## Description
Implementar fluxo de criação de credenciais restrito a administradores, removendo qualquer possibilidade de cadastro público direto no dashboard.

## Requirements
- [x] Não deve existir rota pública de signup.
- [x] Criação/convite de usuário deve exigir sessão autenticada e papel `admin`.
- [x] Ação de provisionamento deve acontecer server-side.
- [x] Deve haver trilha de auditoria para criação de usuário e tentativa negada.
- [x] Fluxo deve suportar reset inicial de senha via mecanismo seguro (Supabase invite/reset).

## Implementation Plan
- Definir endpoint server-side para convite/criação de usuário: `/api/admin/users/invite`.
- Validar sessão por access token e autorização admin via allowlist `AUTH_ADMIN_EMAILS`.
- Integrar com Supabase Admin API (`supabaseServer.auth.admin.inviteUserByEmail`) apenas no backend.
- Registrar eventos de sucesso/falha/negação no audit log de autenticação.
- Manter fase 1 via endpoint (sem UI administrativa ainda).

## Verification
- [x] Usuário sem sessão recebe `401`.
- [x] Usuário autenticado sem role `admin` recebe `403`.
- [x] Admin consegue convidar usuário com sucesso.
- [x] Usuário convidado consegue concluir acesso pelo fluxo definido.
- [x] Logs de auditoria são emitidos para sucesso e negação.
