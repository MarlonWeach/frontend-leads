## Requirements
- [x] Exigir identidade válida nas APIs sensíveis.
- [x] Retornar `401` para requisições não autenticadas.
- [x] Preservar fluxo especial de `/api/sync` baseado em token Bearer.
- [x] Não aplicar autenticação em `/api/auth/*`.

## Implementation Plan
- Definir lista explícita de prefixos de APIs protegidas.
- Validar cookie de sessão no middleware antes do processamento da rota.
- Retornar `401` com payload padronizado quando não autenticado.
- Manter regras existentes de rate limit e Bearer token em `/api/sync`.
