# 25-20 Auto-stub de adsets ausentes no sync de ADS

## Description
Adicionar lógica no script `scripts/sync-ads-once.js` para, antes de salvar os anúncios no Supabase, verificar quais `adset_id` não existem na tabela `adsets` e realizar um upsert mínimo desses adsets a partir da Meta API. Isso evita falhas por restrição de chave estrangeira (`ads_adset_id_fkey`).

## Status History
- InProgress: criado e iniciado em 2025-10-30
- Done: concluído em 2026-04-15 (auto-stub validado e logs consolidados adicionados)

## Requirements
- Verificar `adset_id` dos ads coletados vs. `adsets.id` no Supabase
- Para IDs ausentes: buscar `id,name,status,effective_status,campaign_id,created_time,start_time,end_time` na Meta API e upsertar
- Manter delays e limites atuais para não agravar rate limit
- Não alterar o fluxo de coleta de insights; somente garantir a existência do adset

## Implementation Plan
- Em `scripts/sync-ads-once.js` dentro do fluxo principal:
  - Após obter `activeAds`, montar conjunto de `adset_id`
  - Buscar existentes no Supabase
  - Calcular faltantes; para cada faltante:
    - GET na Meta API e upsert no Supabase
  - Prosseguir com `saveAdsToSupabase`
